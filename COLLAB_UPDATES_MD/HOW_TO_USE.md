# Collab Update Log - How To Use

## Purpose
- Give every collaborator a fast summary of the latest implemented changes.
- Help track what changed without reading the whole codebase or full git history first.

## Folder Rules
- Keep this folder Markdown only.
- Keep updates short, practical, and written for collaborators.

## Workflow
1. After an important gameplay, UI, or systems pass, open `LATEST_UPDATES.md`.
2. Copy the template from `UPDATE_TEMPLATE.md`.
3. Paste the new entry at the top of `LATEST_UPDATES.md`.
4. Fill in date, time, author, branch, summary, files, and notes.
5. Commit and push with the related code changes.

## Recommended Style
- Focus on implemented changes, not intentions.
- Mention important balancing or UX effects.
- Mention risky areas or things that still need verification.
